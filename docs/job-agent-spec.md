# 通用求职 Agent 规格

## 目标

用户配置候选人画像和岗位搜索条件后，Agent 自动读取 BOSS 直聘当前查询条件下的全部岗位结果。Agent 必须跨页或持续滚动到结果耗尽，逐个打开详情页，提取并校验完整岗位信息，保存原始 JD，并生成结构化摘要和候选人匹配分析。

100+ 是系统必须支持的处理规模，不是停止条件。停止条件是当前查询结果已经耗尽，或用户主动停止，或 BOSS 要求用户处理登录/验证码。

## 输入

- 搜索条件：职位关键词、城市、薪资范围、经验、学历、工作类型等。
- 候选人画像：学历、工作年限、技能关键词、目标职位、期望城市、薪资范围、实习/远程偏好、必须条件和可放宽条件。
- 运行策略：是否保存不完全岗位、失败重试次数、请求间隔和是否自动打开详情页。

## 采集流程

```text
启动任务
  -> 登录/验证码预检
  -> 打开搜索结果
  -> 收集当前页面岗位
  -> 自动滚动或翻页直到没有新增岗位
  -> 规范化链接并去重
  -> 逐个打开详情页
  -> 展开完整 JD 并等待内容就绪
  -> 提取字段和原始 JD
  -> 校验字段来源与详情完整度
  -> 保存成功或记录明确失败原因
  -> 继续下一个岗位
  -> 查询结果耗尽后完成
```

## 停止条件

- 搜索结果已经没有下一页或连续多个采集周期没有新增详情链接。
- 用户主动停止。
- 登录、验证码或安全验证需要用户处理，此时任务进入暂停状态，不绕过验证。
- 不因达到 100 条岗位而停止。

## 岗位数据契约

每条成功岗位必须包含：

- `title`：岗位名称。
- `company`：公司名称。
- `city`：工作地点。
- `salary`：薪资原文和可比较的标准化范围。
- `experience`：经验要求。
- `education`：学历要求。
- `link`：经过 BOSS 域名和详情路径校验的详情链接。
- `detailText`：详情页原始可见 JD 文本。
- `jdSummary`：结构化摘要。
- `detailCompleted`：详情完整度布尔值。
- `collectWarnings`：缺失、截断、异常和校验警告。

明确的原始值，例如“面议”“学历不限”“经验不限”，属于有效值。页面没有展示时必须标记为“未展示”，不得猜测或用列表信息冒充详情信息。

## 详情成功标准

- 详情 URL 已成功打开并保持在允许的 BOSS 域名。
- 页面不是登录页、验证码页、失效页或错误页。
- JD 容器已经出现并完成动态加载。
- “查看更多”等展开控件已经处理；仍然截断时必须标记为不完整。
- 至少提取到岗位职责或任职要求中的一类原始内容。
- 详情字段与岗位链接、岗位标题或公司名称至少有可验证的一致性信号。

## 结构化摘要

摘要必须包含：岗位概览、核心职责、硬性要求、加分项、技术栈、经验学历、地点薪资、JD 风险或未知信息、候选人匹配度、匹配理由和能力缺口。

## 任务状态

任务状态必须持久化，至少包括 `runId`、`phase`、`criteria`、`candidateProfile`、`queue`、`cursor`、`processed`、`detailsCompleted`、`saved`、`failed`、`skipped`、`currentJob`、`retryCount`、分页访问记录和 `updatedAt`。

状态包括：`idle`、`preflight`、`collecting_list`、`queue_ready`、`opening_detail`、`waiting_detail`、`extracting`、`filtering`、`saving`、`paused_auth`、`paused_captcha`、`retrying`、`stopped`、`completed`、`error`。

单个岗位失败不得中断整个任务。每个失败岗位都必须保存失败阶段、错误原因、重试次数和详情链接；薪资筛选跳过的岗位也要保留跳过原因。

## 非目标

- 不绕过登录、验证码或安全验证。
- 不自动投递、自动聊天或自动发送消息。
- 不把搜索卡片数据伪装成完整详情数据。
- 不为凑够数量生成虚假字段或虚假 JD。

## 验收标准

- 一个查询任务能够处理 100 条以上岗位，并继续处理直到结果耗尽。
- 100% 的成功记录都有详情链接和原始 JD。
- 详情失败岗位不会进入“详情成功”计数。
- 单个岗位失败不会终止后续采集。
- Service Worker 重启后可以根据持久化状态继续任务。
- 登录或验证码出现时任务暂停，并能在用户处理后继续。
- 任务结束显示查询岗位总数、成功详情数、失败数、跳过数、缺失字段数和保存数。
- 单元测试、集成测试和受控浏览器测试通过；真实 BOSS 小规模验证必须在用户已登录且自行处理验证码的浏览器会话中执行，不绕过平台验证。
